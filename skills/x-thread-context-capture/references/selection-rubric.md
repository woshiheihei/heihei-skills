# Selection rubric

Use this rubric to decide what belongs in the final deliverable file. The goal is context completeness for downstream models, not exhaustive scraping.

## P0: always include

- Root post text and metadata
- Author self-thread entries
- Author clarifications or follow-up explanations
- Author replies that materially refine the root post

## P1: include first after author coverage

- High-profile or verified commenters
- Comments the author replied to
- Comments that attract unusually strong interaction signals
- Comments that become local hubs for follow-on discussion

## P2: include selectively when they add signal

- New facts, examples, or evidence
- Strong objections or reframings
- Useful implementation detail
- Historical, technical, or market background
- Posts containing important X-internal references or meaningful third-party links

## P3: usually exclude

- Pure praise or agreement
- Repeated restatements
- Meme replies without new information
- Off-topic debate

## Author-first rule

If forced to choose, sacrifice breadth of comments before sacrificing author material.

## Reference handling rule

- For X-internal references: capture the original text directly when accessible.
- For third-party pages: keep the URL and the visible preview text unless the user explicitly wants deeper expansion.

## Stop rule

Stop when:

- the root post is fully reconstructed,
- visible author follow-ups are covered,
- the main interpretive branches in comments are represented,
- the important references are preserved, and
- newly loaded comments are mostly repetitive or low-signal.

## Coverage note rule

If X only exposed a partial view, say that plainly. A transparent boundary note is better than pretending completeness.

## Cleanup rule

Once the deliverable has been saved and the coverage note is complete, close the X tabs opened during the task. Leave unrelated pre-existing user pages alone.
