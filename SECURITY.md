# Security Policy

## Scope

This repository contains prompt assets, documentation, and lightweight
validation tooling for a Codex skill. Security issues can still matter when
they affect:

- repository integrity
- unsafe automation guidance
- accidental exposure of secrets or personal data
- vulnerable helper scripts or workflow configuration

## Reporting a vulnerability

Please do not open a public issue for a suspected security problem.

Use GitHub's private vulnerability reporting for this repository if it is
enabled. If private reporting is not available, contact the maintainer through
the repository owner's public GitHub profile contact channel and share only the
minimum details needed to establish contact.

When reporting, include:

- a short description of the issue
- affected file paths
- reproduction steps or proof of concept
- potential impact
- any suggested mitigation

## Disclosure expectations

- give the maintainer reasonable time to investigate and prepare a fix
- avoid publishing exploit details, secrets, or private data in public channels
- note clearly if the issue involves third-party content or platform-policy
  risk
